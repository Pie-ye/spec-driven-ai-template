# Trellis workflow

Use `.trellis/specs/` for durable product/engineering context, `.trellis/research/` for investigation, `.trellis/decisions/` for accepted architecture decisions, `.trellis/prds/` for executable scope, `.trellis/reviews/` for independent verdicts, and `.trellis/retrospectives/` for learning.

Research informs decisions but does not silently change a PRD. A PRD owns acceptance criteria and evidence. The legacy `.trellis/spec/` and `.trellis/tasks/` paths remain only for compatibility with older Trellis tooling.
